OVERVIEW

What changes when a frozen VLA becomes a primitive?

Frozen VLAs are strong at local contact-rich manipulation, but brittle under semantic retargeting, goal re-binding, spatial-layout shifts, long-horizon composition, and unstable contacts. Harness VLA keeps the VLA frozen and exposes it as VLA_ACT, a retryable contact-rich primitive inside a memory-guided agentic loop.

The planner reads the task, live RGB-D observations, robot state, Task Specific Memory, and Global Memory, then emits structured calls to a fixed primitive library. Analytic primitives handle re-grounding, staging, transport, navigation, gripper control, and release, while VLA_ACT is reserved for local contact-rich phases such as grasping, constrained placement, fixture actuation, drawer manipulation, and insertion.

METHOD

How Harness VLA Works

Harness VLA turns a frozen VLA into a retryable contact-rich primitive inside a memory-guided agentic loop. The planner reads language, RGB-D observations, robot state, and memory, then composes VLA_ACT with fixed analytic primitives rather than expanding the skill library at deployment time.

01 Observe

Read the task description, live RGB-D observations, and robot proprioception to recover the current scene and robot state.

02 Retrieve memory

Task Specific Memory provides reusable primitive traces, while Global Memory supplies success rules and failure models.

03 Compose primitives

The planner selects from a fixed primitive library: VLA_ACT for contact-rich execution and analytic primitives for motion, pose, gripper, and navigation control.

04 Execute, verify, retry

Each primitive executes until its post-condition returns control. If contact fails, the planner re-stages the robot and invokes VLA_ACT again.

Bootstrapping

On a reference seed, the planner explores staging orders, pre-contact poses, VLA invocation timing, and stop predicates. Successful primitive programs are serialized as reusable JSONL traces.

Deployment

During evaluation, RESET is disabled. The planner retrieves the stored trace, re-grounds it from live RGB-D observations, applies global rules, and executes through the same fixed primitive interface.

Why It Works

The source paper analyzes three mechanisms: semantic re-grounding, planner-staged VLA invocation, and analytic decomposition around contact-rich control.

Planner-level semantic re-grounding restores task-conditioned behavior

In perturbed LIBERO-Pro cells, RLinf may repeat the standard behavior even when the instruction or layout changes. Harness VLA re-grounds targets through the planner.
