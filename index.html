OVERVIEW

Learning to use frozen VLAs through memory-guided execution

Vision-language-action models already provide strong local manipulation skills, but their reliability drops when instructions, objects, layouts, or task horizons change. Harness VLA addresses this gap without changing the VLA weights: it learns a harness around the frozen policy, turning each contact-rich action into an attempt that can be staged, checked, diagnosed, and retried.

The central idea is not to enlarge the skill library, but to learn how a small fixed set of primitives should be used. During bootstrapping, the planner searches over staging orders, pre-contact poses, VLA_ACT timing, stop predicates, and recovery choices. Successful traces become Task Specific Memory, while reusable success rules and failure models become Global Memory. At deployment, the planner transfers this learned structure by re-grounding objects, fixtures, and target poses from the current RGB-D observation, rather than replaying reference-scene coordinates.

METHOD

How Harness VLA Learns

Harness VLA exposes a frozen VLA as VLA_ACT, but the learning happens in the harness. On a reference seed, the planner explores how the fixed primitives behave, records what worked, keeps useful failures as negative evidence, and abstracts the resulting strategy into memory. In evaluation, it reads the task, live RGB-D observations, robot state, Task Specific Memory, and Global Memory, then re-grounds the learned primitive structure for the current scene.

Learning primitive operating ranges under perturbations

What the harness learns

  • Which subproblems should be handled analytically, and which require VLA_ACT.
  • Which pre-contact poses, prompts, stop predicates, and retry patterns make VLA_ACT reliable.
  • Which failure signatures, such as empty grasps or false visual success, should trigger re-grounding.

The stored trace is a reusable solution skeleton, not an open-loop trajectory: deployment keeps the learned primitive order but binds spatial arguments from the live scene.

Closed-loop learning loop

01 Observe

Read the task, RGB-D observations, robot state, benchmark signal, and execution logs.

02 Try

Select one primitive call and test a staging choice, VLA_ACT timing, or stop condition.

03 Diagnose

Classify the outcome as progress, recoverable failure, or unrecoverable failure.

04 Write memory

Store successful structure, useful recovery decisions, and failure models for later use.

Two-phase memory workflow

Bootstrapping

On one reference seed, reset is available and the planner searches over staging orders, pre-contact poses, prompts, VLA_ACT timing, stop predicates, and recovery decisions. Later successful attempts can replace earlier traces, while failed attempts remain as constraints and failure models.

Deployment

Reset is disabled and the budget is strict. The planner retrieves the learned task skeleton and global operating rules, then re-localizes every object, fixture, support surface, and target pose from live observations before issuing primitive calls.

ANALYSIS

Why Harness Learning Works

Harness VLA improves reliability because exploration teaches the planner how to use a fixed primitive set: where to ground entities, when to invoke VLA_ACT, how to re-stage failed contacts, and which failures should not be repeated.

Planner-level semantic re-grounding restores task-conditioned behavior

In perturbed LIBERO-Pro cells, Frozen VLA may repeat the standard behavior even when the instruction or layout changes. Harness VLA uses the learned task skeleton only as structure, then re-grounds the target from the current observation.

Planner-staged VLA invocation improves frozen-policy reliability

Bootstrapping teaches where VLA_ACT should be attempted and how it should be re-staged after brittle contacts. A small number of planner-selected calls rapidly lifts success above the frozen-policy baseline.

Success unlocked by a small VLA-call budget

Cumulative success as the maximum number of VLA_ACT invocations per episode increases. Dashed lines mark the frozen-policy baseline and full Harness VLA performance.

Analytic primitives isolate non-contact execution

The harness learns a division of labor: analytic primitives handle reproducible geometry such as transport, staging, retreat, and release, while VLA_ACT is reserved for local contact-rich phases.

Primitive attribution across successful rollouts

Bars show which primitive class triggers the final success predicate in solved tasks.

Successful rollouts finish through different primitive classes depending on task structure: LIBERO-style tasks often complete after analytic transport or release, while RoboCasa365 and RoboTwin C2R more often finish inside contact-rich VLA_ACT phases.
