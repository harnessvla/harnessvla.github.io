Memory-guided agentic manipulation

Harness VLA: Steering Frozen VLAs into Reliable Manipulation Primitives via Memory-Guided Agents

Harness VLA embeds a frozen VLA into a memory-guided agentic loop. The planner retrieves successful traces and failure rules, then composes VLA_ACT with fixed analytic primitives to re-ground, re-stage, retry, and complete long-horizon manipulation tasks.

OVERVIEW

What changes when a frozen VLA becomes a primitive?

Frozen VLAs are strong at local contact-rich manipulation, but brittle under semantic retargeting, goal re-binding, spatial-layout shifts, long-horizon composition, and unstable contacts. Harness VLA keeps the VLA frozen and exposes it as VLA_ACT, a retryable contact-rich primitive inside a memory-guided agentic loop.

The planner reads the task, live RGB-D observations, robot state, Task Specific Memory, and Global Memory, then emits structured calls to a fixed primitive library. Analytic primitives handle re-grounding, staging, transport, navigation, gripper control, and release, while VLA_ACT is reserved for local contact-rich phases such as grasping, constrained placement, fixture actuation, drawer manipulation, and insertion.

METHOD

How Harness VLA Works

Harness VLA exposes a frozen VLA as VLA_ACT, a retryable contact-rich primitive inside a memory-guided agentic planner. At each turn, the planner reads the task, RGB-D observations, robot state, Task Specific Memory, and Global Memory, then emits a structured call to a fixed primitive library rather than low-level robot actions.

Primitive composition under deployment perturbations

Why the harness helps

  • Deployment perturbations move targets beyond the direct VLA rollout distribution.
  • Analytic primitives re-ground task bindings and route motion through shifted layouts.
  • VLA_ACT is invoked only for contact-rich phases inside VLA-compatible local regions.

Primitive composition lets Harness VLA traverse semantic retargeting and spatial shifts with analytic control, while reserving VLA_ACT for local contact-rich regions.

Execution loop

01 Observe

Read the task, RGB-D observations, and robot state.

02 Retrieve

Load task traces plus global success rules and failure models.

03 Compose

Emit one structured primitive call: VLA_ACT or an analytic primitive.

04 Verify & retry

Return after the primitive post-condition; re-stage and retry failed contacts.

Two-phase memory workflow

Bootstrapping

On one reference seed, the planner explores staging orders, pre-contact poses, VLA_ACT timing, and stop predicates. Successful primitive programs are stored as reusable JSONL traces, while general success rules and failure models are written into Global Memory.

Deployment

RESET is disabled. The planner retrieves the stored trace, re-grounds symbolic targets from live RGB-D observations, applies Global Memory, and executes through the same fixed primitive interface.

ANALYSIS

Why It Works

Harness VLA improves reliability by re-grounding task entities under perturbations, invoking VLA_ACT only for sparse contact-rich phases, and using analytic primitives for staging, transport, and recovery.

Planner-level semantic re-grounding restores task-conditioned behavior

In perturbed LIBERO-Pro cells, RLinf may repeat the standard behavior even when the instruction or layout changes. Harness VLA re-grounds targets through the planner.

Planner-staged VLA invocation improves frozen-policy reliability

A small number of planner-selected VLA_ACT calls rapidly lifts success above the frozen-policy baseline and approaches full Harness VLA performance.

Success unlocked by a small VLA-call budget

Cumulative success as the maximum number of VLA_ACT invocations per episode increases. Dashed lines mark the frozen-policy baseline and full Harness VLA performance.

Analytic primitives isolate non-contact execution

Analytic primitives handle transport, staging, base/wrist adjustment, retreat, and release, while VLA_ACT remains responsible for contact-rich local phases.

Primitive attribution across successful rollouts

Bars show which primitive class triggers the final success predicate in solved tasks.

Successful rollouts finish through different primitive classes depending on task structure: LIBERO-style tasks often complete after analytic transport or release, while RoboCasa365 and RoboTwin C2R more often finish inside contact-rich VLA_ACT phases.
