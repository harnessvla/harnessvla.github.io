Language-conditioned manipulation requires both precise contact-rich control and robust reasoning over language, scenes, and long horizons. End-to-end VLA models provide strong local visuomotor skills, but often fail under semantic retargeting, goal re-binding, spatial-layout shifts, and unstable local contacts.

Harness VLA exposes a frozen VLA as a retryable contact-rich primitive and composes it with a fixed analytic primitive library for grounding, staging, transport, navigation, and release. The harness learns the operating range of these primitives from task-specific traces, global success rules, and failure models.

The central claim is asymmetric delegation: the VLA remains a local contact specialist, while the planner owns task binding, non-contact execution, and recovery after failed or incomplete local attempts.

How Harness VLA Works

The planner conditions on task language, RGB-D observations, proprioception, Task Specific Memory, and Global Memory. It emits structured calls to a fixed primitive library rather than expanding the primitive set at deployment time.

01

Freeze the VLA as a contact specialist

VLA ACT is the only learned primitive. It is invoked for grasping, constrained placement, fixture actuation, insertion, and other local contact-rich phases.

02

Keep the action interface fixed

The planner emits JSON calls to predefined VLA, composite analytic, and atomic analytic primitives. It never writes new deployment-time skills or low-level actions.

03

Use memory to re-ground and retry

Task traces preserve the solution skeleton, while Global Memory records success rules and failure models such as empty grasps and false completion checks.

Bootstrapping

Explore on a reference seed

RESET is available only during exploration. The planner tries staging orders, pre-contact poses, VLA invocation timing, and stop predicates until it finds a successful primitive program.

Memory

Store a reusable solution skeleton

Task Specific Memory stores a JSONL trace and summary. Concrete coordinates are treated as reference bindings, so deployment must re-ground objects and targets from live RGB-D observations.

Evaluation

Deploy without reset or new skills

During formal evaluation, RESET is disabled and the budget is shorter. The planner retrieves the trace, uses Global Memory, and executes through the same fixed primitive interface.

Why It Works

The source paper analyzes three mechanisms: semantic re-grounding, planner-staged VLA invocation, and analytic decomposition around contact-rich control.

Planner-level semantic re-grounding restores task-conditioned behavior

In perturbed LIBERO-Pro cells, RLinf may repeat the standard behavior even when the instruction or layout changes. Harness VLA re-grounds targets through the planner.

Planner-staged VLA invocation improves frozen-policy reliability

Conclusion and Limitations

Harness VLA casts a frozen VLA as a single contact-rich primitive interface within an LLM-driven agent, delegating transport, posture, navigation, and release phases to the planner.

Open feedback loop

The current framework is limited by an open feedback loop between the high-level planner and low-level VLA.

No joint fine-tuning

The system lacks joint fine-tuning via environmental rewards and human preferences.

Captioning bottleneck

The absence of fine-grained image captioning constrains structural reasoning in highly cluttered, long-horizon tasks.
